# 项目介绍

Codex Cleaning Temporary Processes 是一个公开的跨平台包，用于安全清理临时开发进程。

这个包支持两种安装模式：

- 插件式安装：在仓库根目录保留 `.codex-plugin/plugin.json`、`hooks.json` 和 `hooks/`，从而启用自动强触发
- skill 式安装：放到 `CODEX_HOME/skills` 下，只暴露 skill 文本和脚本，本身不会自动启用 hook 行为

这种区分是面向用户、而且是刻意设计的。已安装的插件包负责固定检查点自动触发，`SKILL.md` 则继续作为手动兜底指南。

固定检查点模型也是刻意设计的。它不会等 Codex “快到任务结尾时再想起清理”，而是在明确结束的检查点之后重新评估，例如高风险工具步骤结束、DevTools 或浏览器调试检查点结束、子代理结束、一次性 helper 积压出现之后，以及会话结束时。

整体安全模型保持收敛：

- 证据不足时先 `inspect`
- checkpoint 阶段只清理高置信度残留
- 显式自动化必须有当前任务链路或当前线程确认归属
- 仅有工作区匹配仍然不够
- 当前线程拥有权不会放宽普通 runtime 的清理范围
- 保留活动中的 Codex shell、普通用户应用、存在歧义的 runtime，以及可能仍可复用的开发服务

这个包还把多项目隔离写得很明确。带工作区归属的 runtime 清理仍然必须匹配当前工作区或当前任务祖先进程；来自其他对话或其他仓库的显式自动化，在归属没有被证明之前继续保持 `inspect-only`。

对于同线程显式自动化恢复，本地运行时状态会使用经过清理的线程标识和规范化后的工作区值。这些本地状态只用于安全查找和文件命名，不会扩大清理权限。

这个仓库包含公开文档、元数据、插件清单、hook 定义、PowerShell 脚本、shell wrapper 以及测试，用于在 Windows、macOS 和 Linux 上打包这套行为。
