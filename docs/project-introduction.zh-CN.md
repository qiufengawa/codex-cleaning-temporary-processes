# 项目介绍

Codex Cleaning Temporary Processes 是一个面向公开发布的、以 Windows 为重点的 Codex skill，用来安全地管理开发过程中的临时进程卫生。

它适用于这类场景：包管理器、开发服务器、测试命令、浏览器自动化、DevTools MCP 辅助进程以及工作区相关运行时，都会在任务过程中留下临时进程。这个 skill 不要求你等到整个长任务结束后再统一清理，而是支持在某个高风险步骤结束后做 checkpoint cleanup，从而控制进程堆积，同时避免“看起来像开发进程就全部杀掉”的粗暴做法。

这个项目遵循保守的安全策略：

- 证据不足时先检查，不直接清理
- checkpoint 阶段只清理高置信度、已经没有复用价值的残留
- 保留当前 Codex shell、普通用户应用和可能仍会复用的开发服务
- 公开示例保持脱敏，不包含私有工作区细节

这个仓库包含了可复用的 skill 定义、agent 元数据、PowerShell 脚本和 Pester 测试，方便在其他 Codex 环境中直接使用或继续扩展。
